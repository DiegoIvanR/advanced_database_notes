# Session – 2026-02-17

## Topics covered
- Joins: Types (Inner, Left, Right, Full), Join Algorithms (Nested Loops, Hash, Sort Merge), and Optimization.

- Aggregations: Grouping data, Aggregate Functions (SUM, AVG, COUNT), and the HAVING clause.

- Indexing Fundamentals: B-Tree structures, Selectivity, and Performance trade-offs.

- Query Optimization: Scan types, Execution plans, and the "Golden Rules" of SQL performance.

## What I understood
- Effective indexing requires high selectivity and "SARGable" queries that avoid wrapping columns in functions to prevent Full Table Scans. For Joins and Aggregations, performance is optimized by indexing Foreign Keys and using WHERE to filter data before it hits the GROUP BY or HAVING stages. Ultimately, the SQL Optimizer relies on fresh statistics to choose between fast index lookups or memory-intensive table reads.
