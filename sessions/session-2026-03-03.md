## Topics covered
- Window function syntax: `OVER()`
- `PARTITION BY` vs `GROUP BY`
- `ORDER BY` inside window functions
- Ranking functions: `ROW_NUMBER`, `RANK`, `DENSE_RANK`
- Aggregate window functions (running total, partition total)
- `LAG` and `LEAD`
- Filtering window results using subqueries / CTEs

## What I understood
- window functions let us perform calculations across related rows without collapsing them
- unlike group by, they keep every original row visible
- partition by splits data into logical groups, but does not reduce rows
- order by inside over() controls ranking or running calculations
- row_number gives unique numbering
- rank gives same rank to ties but skips numbers
- dense_rank gives same rank to ties without skipping numbers
- sum() over(partition by ...) gives total per group while keeping row-level detail
- window function aliases cannot be used directly in where; need subquery or CTE
- DENSE_RANK!!!!
