## What I understood

- A transaction is a group of operations that must succeed or fail together  
- In Oracle, a transaction starts automatically with the first `INSERT`, `UPDATE`, or `DELETE`  
- Transactions end explicitly with `COMMIT` (save changes) or `ROLLBACK` (undo changes)  
- `SAVEPOINT` allows partial rollback within a transaction  
- You should verify data before committing, especially in manual workflows  

- **Atomicity** → all or nothing  
- **Consistency** → data stays valid  
- **Isolation** → transactions don’t interfere with each other  
- **Durability** → committed changes are permanent  


- Functions return a value and can be used in `SELECT` statements  
- Procedures perform actions and cannot be used in `SELECT`  
- Functions should not modify data when used in queries  

- Removes control from the caller  
- Can cause partial commits in larger transactions  


- Should not be inside transactions  
- Because they are not part of database consistency
